# X Thread Template

Apply this template when generating X (Twitter) distribution artifacts. The structure reflects what performs on X in 2026 for operator-researcher positioning, not growth-hacker thread patterns.

## Structural template

```
1/ [HOOK — specific observation that generates curiosity without clickbait]

2/ [SETUP — the problem or condition the framework addresses]

3/ [FRAMEWORK — named concept stated explicitly with brief definition]

4/ [MECHANISM — how the framework actually works in practice]

5/ [RECEIPT — concrete evidence, precedent, or quantified example]

6/ [IMPLICATION + essay link]
```

## Tweet budget

- Each tweet: 240–270 characters
- Thread length: 5–6 tweets, never more
- Hard maximum: 7 tweets only if the framework genuinely cannot be compressed

Resist the temptation to extend. A tight 5-tweet thread outperforms a sprawling 10-tweet thread on engagement, citation, and reshare metrics.

## Formatting requirements

- **Number each tweet** with "N/" prefix — this is the load-bearing structural signal on X
- **Em-dashes rather than semicolons** for mid-sentence pivots
- **No emoji** — violates voice markers
- **No hashtags** — violates voice markers
- **Essay URL in final tweet only** — X's algorithm penalizes early-tweet outbound links
- **Never use "thread" as self-reference** (e.g., "A thread 🧵" is prohibited)

## Generation rules specific to X

1. **Tweet 1 is the only one that matters for reach**. X's algorithm weights first-tweet engagement heavily for thread distribution. Invest disproportionate attention on Tweet 1's specificity and hook quality.

2. **Each tweet must stand alone**. A reader who sees only Tweet 3 in their feed should understand the concept. Avoid "as I mentioned above" or back-references that require full-thread reading.

3. **Tweet 3 must name the framework explicitly**. This is the tweet that gets screenshot-shared and quote-tweeted. Make the framework name and core definition unmistakable.

4. **Tweet 5 grounds in specifics**. Abstract threads lose engagement in the middle. A concrete receipt (metric, case study, precedent) in the evidence tweet reanchors reader attention.

5. **Tweet 6 must include the essay link, nothing else**. No additional CTAs, no "follow for more," no "subscribe to my newsletter." The essay link does the work.

## What to avoid

Do not write X threads that:

- Open with "🧵" or "🧠" or any thread-indicator emoji
- Use "Here's a thread on..." or "Let me explain..." openings
- Include "bookmark this" or "save this thread" language
- Use numbered lists within tweets (the thread itself is the numbered list)
- End with "follow @briantighe for more" or similar follow-fishing
- Include "quote tweet if you agree" or engagement-fishing prompts

## Handling the 280-character constraint

When compressing essay content into 240–270 characters, apply this prioritization:

1. **Keep the specific and concrete**; cut the general and abstract
2. **Keep precedents and named entities**; cut hedges and qualifiers
3. **Keep the framework name**; cut synonyms and restatements
4. **Keep the receipt's metric**; cut the receipt's context if forced

If a tweet exceeds 270 characters after compression, the content is too dense — split across two tweets rather than cut below the minimum viable concept.

## Voice check before finalizing

Apply the test: would Ben Thompson's Stratechery audience, scrolling X, recognize this as analytically serious content? If the thread reads as growth-hacker cadence or Twitter-thread-style guru positioning, rewrite.

The register is closer to Simon Willison's weblog tweets, Byrne Hobart's analytical threads, or the Stratechery account's measured distribution than to any standard "guru thread" account. Error toward analytical seriousness over punchiness.
