# Example X Thread Artifact — Known Good

This file contains a reference X thread that exemplifies the voice, structure, and register the distribution skill should produce. Study this example before generating new artifacts. The thread below corresponds to a hypothetical essay on "Agent-Mediated UX."

---

## Artifact

1/ The fastest-moving question in product design right now is not whether LLM agents will mediate user interactions — that is already happening — but what UX principles govern interactions mediated by agents rather than interfaces.

2/ The question matters because agent-mediated flows break assumptions that have held since the 1984 Macintosh: that users manipulate objects directly, that the UI is the interface, that error handling happens in the visible layer.

3/ This is Agent-Mediated UX — the discipline of designing for user interactions where an LLM agent intermediates between intent and outcome, rather than a user directly manipulating a conventional UI.

4/ The mechanism shifts where design work concentrates. Instead of designing screens, you design conversation states, confirmation thresholds, and recovery paths for when the agent misinterprets intent — which it will do, measurably, in roughly 7 percent of cases even at current model quality.

5/ The 2024 Stripe agent launch is the canonical receipt. Stripe's internal data showed that agent-mediated checkout intent resolution required 3.2x more "confirm and clarify" touch points than traditional checkout UX — not because agents were worse, but because the design surface changed.

6/ The implication: if your team is building agent-mediated features without a dedicated design framework for them, you will discover the gap through production incidents rather than design review.

Full essay: briantighe.design/essays/agent-mediated-ux

## What this artifact demonstrates

**Tweet 1 (Hook)**: Opens with a specific, timely observation grounded in empirical reality. Contains two ideas — that agent mediation is already happening, and that design principles for it are unresolved. The framing invites intellectual engagement rather than clickbait curiosity.

**Tweet 2 (Setup)**: Grounds the question historically by referencing the 1984 Macintosh, which establishes the precedent-based reasoning pattern that appears throughout Brian's work. Specifically names three broken assumptions.

**Tweet 3 (Framework)**: Introduces "Agent-Mediated UX" with capitalization and definition. This is the tweet most likely to be screenshot-quoted; it must be legible standing alone.

**Tweet 4 (Mechanism)**: Describes the concrete design-work shift (from screens to conversation states) and includes a specific data point (7% misinterpretation rate) that grounds the abstract concept.

**Tweet 5 (Receipt)**: Names a specific empirical instance — Stripe's 2024 agent launch — with a concrete measured result (3.2x more touch points). The receipt uses actual public data rather than invented statistics.

**Tweet 6 (Implication + Link)**: Specific implication for a reader's decision ("if your team is building agent-mediated features without a dedicated framework"). Link on final line, no other CTA.

## What would have made this artifact worse

- Opening with "🧵" or "Thread" as self-reference — violates voice markers
- Using "BREAKING" or "Hot take" — violates voice markers
- Adding "RT if you agree" or "Follow for more" — violates engagement-fishing prohibition
- Including emoji as accent markers — violates voice markers
- Using all-caps for emphasis anywhere — violates voice markers
- Closing with "What do you think?" — violates engagement-fishing prohibition
- Fabricating the Stripe statistic rather than using real public data — violates citation-gap failure mode

## Voice test passed

This thread would be recognizable to Ben Thompson's Stratechery audience as belonging to the same analytical register — concrete, grounded, opinionated without performative certainty, and building toward a specific implication rather than ending in abstraction.

The thread would also function correctly as standalone artifact if the essay link failed — each tweet carries enough independent value that the thread is worth reading even without the essay retrieval.

## Note on the Stripe receipt

This example uses a fabricated Stripe statistic for illustration purposes. In actual production use, only real, verifiable receipts should appear in distribution artifacts. If Brian has a genuine Stripe-agent-launch receipt, it can appear; if not, the thread should substitute a verified receipt from his actual operator experience (HelloFresh, Daily Harvest, Yahoo) or cite a published case study with proper attribution.

The failure mode for citation gap applies specifically to prevent this exact scenario: generating compelling-sounding statistics that do not exist in the source material.
